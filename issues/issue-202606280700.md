# 課題解決プロンプト: buildInitScript のテンプレートリテラル混在（String.raw と通常テンプレート）を解消する

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `pi-script.ts` の `buildInitScript` 関数内で、`String.raw` テンプレートリテラルと通常のテンプレートリテラルが混在している：
  - `String.raw` は bash 変数展開（`${HOST_IP}`）を TypeScript に解釈させないために使われている
  - 一方、`export PI_PROVIDER="${cliProvider}"` の部分では TypeScript 補間が必要なため通常のテンプレートリテラルが使われている
  - この混在により `hostIpRef()` / `hostProjectNameRef()` のような回避関数が必要になり、コードの見通しが悪い
- **再現手順**: `pi-script.ts` の `buildInitScript` 関数内の `commonScript`、`exportBlock`、default モードの3つのテンプレートリテラルの使われ方を確認する。
- **関連ファイル/コード**:
  - `pi-script.ts`: `buildInitScript`、`hostIpRef`、`hostProjectNameRef` 関数

## 3. 解決すべきゴール (Goal & Objective)
- [ ] 単一のテンプレート方式に統一する（例: すべて `String.raw` ベースにして TypeScript 補間値を事前に変数化する、またはすべて通常テンプレートにして bash 変数をエスケープする）
- [ ] `hostIpRef()` / `hostProjectNameRef()` ヘルパー関数を削除できるようにする
- [ ] 生成されるシェルスクリプトの内容が変わらないことをテストで確認する
- [ ] この改善は「シェルスクリプトテンプレートの外出し」（issue-202606280656）と競合する可能性があるため、先に issue-202606280656 を解決することが推奨される

## 4. 期待する成果物 (Output Format)
- リファクタリング後の `pi-script.ts` の差分
- テスト実行結果（`npm test` 全30件通過）
