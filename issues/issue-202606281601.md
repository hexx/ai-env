# 課題解決プロンプト: index.ts の重要関数のユニットテスト追加（最深刻：現状ノーテスト）

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `index.ts`（387行）は CLI エントリポイント・クレデンシャル取得・コンテナ起動引数組み立てを担う最重要モジュールであるにもかかわらず、**ユニットテストが1件も存在しない**。結果として、コンテナ起動時のセキュリティクリティカルな処理が未検証のままリリースされている。
- **再現手順**: `find . -name "*.test.ts" | grep -v node_modules` を実行すると `pi-projects.test.ts` のみが見つかり、`index.test.ts` は存在しない。
- **関連ファイル/コード**:
  - `index.ts`: 以下の関数がすべて無テスト
    - `redactSecrets` / `SECRET_ENV_PATTERN`（stderr ログからAPIキー漏洩を防ぐマスク処理。バグがあればクレデンシャル漏洩）
    - `detectProfileName`（パスセグメントからプロファイル名を解決するコアロジック）
    - `buildEnvArgs`（コンテナに渡す環境変数10個の組み立て）
    - `buildVolumeArgs`（`~/.ssh` を `ro` でマウントする設定。`ro` が抜けるとコンテナからホストの鍵が改変可能）
    - `loadCredentials`（Keychain + `gh auth token` からの取得統括）
    - `handleError`（ユーザに見せるエラーメッセージの整形）
    - `isMacOS`、`requireEnv`、`buildContainerArgs`
