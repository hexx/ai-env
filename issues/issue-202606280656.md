# 課題解決プロンプト: シェルスクリプト生成（TypeScript文字列連結）のアーキテクチャ改善

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `pi-script.ts` の `buildInitScript` 関数が、複雑な bash スクリプトを TypeScript のテンプレートリテラル文字列連結で生成している。このため、以下の問題が発生している：
  1. 生成後のシェルスクリプトに syntax error があっても TypeScript コンパイルでは検出できず、デバッグが困難
  2. テストが生成文字列の正規表現パースに依存しており、空白や改行の変更だけで壊れる脆弱性がある
  3. TypeScript テンプレート補間（`${cliProvider}`）と bash 変数展開（`${HOST_IP}`）が同居し、`hostIpRef()` のような回避関数が必要になるなど可読性が低い
  4. シェルインジェクションの潜在的リスク（バリデーションは徹底されているが、漏れた場合の影響が大きい）
- **再現手順**: `pi-script.ts` の `buildInitScript` 関数のコードを読むと、`String.raw` と通常テンプレートリテラルが混在していることが確認できる。
- **関連ファイル/コード**:
  - `pi-script.ts`: `buildInitScript`, `generatePiResumeFunc`, `generateCaseBody` 関数
