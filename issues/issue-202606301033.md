# 課題解決プロンプト: GitHub Actionsでテストを自動実行するCIの導入

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよびCI/CDパイプライン構築のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: 現在、このプロジェクト（`ai-env`）にはGitHub ActionsによるCI（継続的インテグレーション）が一切設定されていません。プッシュやプルリクエスト時に自動でテストが実行されないため、コード品質の継続的な保証ができていません。
- **プロジェクト構成**:
  - Node.js プロジェクト（`engines: ">=20"`）
  - TypeScript 使用（`tsx` で実行）
  - テストフレームワーク: Node.js ネイティブテストランナー（`node --test`）
  - テスト実行コマンド: `node --import tsx --test *.test.ts`
  - テストファイル: `index.test.ts`, `pi-config.test.ts`, `pi-script.test.ts`, `pi-validation.test.ts`
  - ビルドコマンド: `npm run build`（esbuild + assets copy）
  - リンター: `oxlint`（`@types/node`, `esbuild`, `oxlint`, `tsx`, `typescript` が devDependencies）
  - `.github/workflows/` ディレクトリは未作成
- **再現手順**: リポジトリへのプッシュやプルリクエスト作成時に、自動テストが実行されない状態であることを確認

## 3. 解決すべきゴール (Goal & Objective)
- [x] `.github/workflows/ci.yml` を作成し、GitHub Actionsでテストを自動実行できるようにした
- [x] ワークフローは以下のトリガーで実行されること:
  - `main` ブランチへのプッシュ時
  - `main` ブランチに対するプルリクエスト作成・更新時
- [x] Node.js のバージョンは `>=20` に対応し、`22.x`（LTS 最新）を使用する
- [x] `npm ci` で依存関係をインストールし、`npm test` でテストを実行すること
- [x] オプション: リンター（oxlint）の実行ステップも含めた
- [x] 既存の `package.json` の `scripts` を変更しないこと

## 4. 期待する成果物 (Output Format)
- 作成された `.github/workflows/test.yml`（または適切な名前）のワークフロー定義ファイル
- ワークフローの内容と意図を説明する簡潔な解説
- 可能であれば、実際にGitHub上でワークフローが動作するかの確認方法の提示

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、直ちにステップ2の「関連ファイル」の解析から始めてください。
