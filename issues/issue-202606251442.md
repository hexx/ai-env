# 課題解決プロンプト: pi-projects.json で model フィールドにコロン区切りの思考レベル指定を許可する

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `pi-projects.json` で model フィールドに `deepseek-v4-flash:xhigh` のような「モデルID:思考レベル」のコロン区切り書式を記述すると、バリデーションで弾かれる。理由は model 値の検証に `SAFE_SHELL_PATTERN = /^[a-zA-Z0-9._-]+$/u` を使用しており、コロン (`:`) が許可されていないため。
- **再現手順**:
  1. `pi-projects.json` の projects または profiles 内で `"model": "deepseek-v4-flash:xhigh"` と記述する。
  2. `loadAiEnvConfig()` を実行する。
  3. `requireSafeId()` が `SAFE_SHELL_PATTERN` にマッチせずエラーになる。
- **関連ファイル/コード**:
  - `/workspace/pi-projects.ts` — `SAFE_SHELL_PATTERN` 定数 (L53)、`parseProfileOptionalFields` (L176-188)、`parseProjectObjectValue` (L230-253) で model のバリデーションに使用
  - `SAFE_SHELL_PATTERN = /^[a-zA-Z0-9._-]+$/u` (L53) — コロンが含まれていない
  - `generatePiResumeFunc` (L91-118) — model 値は `--model <value>` としてシェル引数に渡されるため、コロンはシェル上で特別な意味を持たず安全
  - `/workspace/pi-projects.test.ts` — model のコロン区切りに関するテストケースが未実装

## 3. 解決すべきゴール (Goal & Objective)
- [ ] model フィールドで `deepseek-v4-flash:xhigh` のような `モデルID:思考レベル` のコロン区切り書式を許容する。
- [ ] コロンを許可する新しいパターン（例: `SAFE_MODEL_PATTERN`）を導入し、model フィールドのバリデーションにのみ使用する。
- [ ] `SAFE_SHELL_PATTERN` 自体は変更せず、project key / session / provider 等の既存バリデーションに影響を与えない。
- [ ] `defaultModel` の値を `--model` フラグ経由で pi に渡す `buildOptionalFlag` や `generatePiResumeFunc` の生成ロジックはコロン入り文字列をそのまま通せており、変更不要であることを確認する。
- [ ] 対応するテストケースを `pi-projects.test.ts` に追加する（model にコロン入り文字列を指定した場合のパース成功／不正文字を含む場合のエラー）。
- [ ] `npm test` がすべてパスすることを確認する。
