現在、`package.json` の `"bin"` に直接 TypeScript ファイル (`index.ts`) が指定されており、シバンが `#!/usr/bin/env -S npx tsx` となっています。
この状態で `npm install -g .` してコマンドを実行すると、実行時のカレントディレクトリの `node_modules` に影響されて動作が不安定になる問題（モジュール解決の汚染）があります。

これを解決するため、TypeScriptファイルを事前にJavaScriptにビルドし、Node.jsで直接実行できる形でインストールされるようにプロジェクトを改修してください。

以下の要件に従って修正を行ってください。

### 1. ビルドプロセスの導入 (`package.json`)
- `devDependencies` に `esbuild` を追加してください。
- `"scripts"` に以下のコマンドを追加してください。
  - `"build": "esbuild index.ts --bundle --platform=node --format=esm --outfile=dist/index.js"`
  - `"prepack": "npm run build"` （npm install -g 時に自動でビルドされるようにするため）

### 2. エントリーポイントの変更 (`package.json`)
- `"bin"` の `"ai-env"` の向き先を `"./index.ts"` から `"./dist/index.js"` に変更してください。

### 3. シバンの修正と付与
- 現在の `index.ts` の先頭にある `#!/usr/bin/env -S npx tsx` を削除するか、`#!/usr/bin/env node` に変更してください。
- `esbuild` のビルドスクリプトでシバンを残すために、`package.json` の `"build"` スクリプトに `--banner:js="#!/usr/bin/env node"` を追加するなどの工夫をしてください。

これらの修正を適用し、安全にグローバルインストール可能なパッケージ構成にしてください。
