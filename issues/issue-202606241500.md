# 課題解決プロンプト: ai-env起動時のデフォルトセッション動作変更

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: ai-env起動時にpi-projects.jsonからセッション情報を自動的に読み込んでしまい、新規セッションを開始しにくい
- **再現手順**: ai-envコマンドをオプションなしで実行すると、既存のセッションが引き継がれる
- **関連ファイル/コード**: ai-envスクリプト（具体的なパスは調査が必要）、pi-projects.json

## 3. 解決すべきゴール (Goal & Objective)
- [ ] ai-env起動時のデフォルト動作を「セッション指定なし（新規セッション）」に変更する
- [ ] 何らかのオプション（例: --session, -s など）を指定した場合のみ、pi-projects.jsonのセッションを引き継ぐようにする
- [ ] 既存の機能を壊さないこと

## 4. 期待する成果物 (Output Format)
- 修正が完了したコード一式、または修正パッチ
- 修正理由とアプローチの簡潔な解説
- 動作確認またはテストの実行結果

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、直ちにステップ2の「関連ファイル」の解析から始めてください。
