現在、`package.json` の `"build"` スクリプトに `--packages=external` オプションが指定されていますが、この設定により `pnpm install -g .` でグローバルインストールした際に、実行時に `commander` モジュールが見つからず `ERR_MODULE_NOT_FOUND` エラーが発生してしまいます。

これを解決するため、CLIツールとして依存関係（`commander` など）もすべて `dist/index.js` に同梱（バンドル）するように修正してください。
`esbuild` は `--platform=node` が指定されていれば Node.js の組み込みモジュールは自動的に外部化してくれます。

### 修正内容
`package.json` の `"scripts"` 内の `"build"` コマンドから、`--packages=external` オプションを削除してください。

変更前:
"build": "esbuild index.ts --bundle --platform=node --format=esm --packages=external --banner:js=\"#!/usr/bin/env node\" --outfile=dist/index.js"

変更後:
"build": "esbuild index.ts --bundle --platform=node --format=esm --banner:js=\"#!/usr/bin/env node\" --outfile=dist/index.js"

この修正を `package.json` に適用してください。
