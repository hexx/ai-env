### pi-coding-agent への拡張指示プロンプト

```markdown
`ai-env` の設定管理を大幅に拡張し、ディレクトリによる「仕事用 / プライベート用」のプロファイル自動判別と、全体設定（OCR関連）の動的切り替え機能を実装してください。
以下の要件に従って `index.ts` と `pi-projects.ts` を改修してください。

### 1. 設定ファイルの構造変更 (`pi-projects.ts`)
これまでの「プロジェクトとセッションのマッピングのみ」の JSON 構造から、プロファイルごとの全体設定も持てる構造に拡張してください。
期待する新しい設定ファイル（JSON）の構造イメージは以下の通りです。

```json
{
  "profiles": {
    "pi-private": {
      "OCR_USE_ANTHROPIC": "false",
      "OCR_LLM_URL": "[https://opencode.ai/zen/go/v1](https://opencode.ai/zen/go/v1)",
      "OCR_LLM_TOKEN_KEY": "OPENCODE_API_KEY",
      "OCR_LLM_MODEL": "mimo-v2.5-pro"
    },
    "pi-work": {
      "OCR_USE_ANTHROPIC": "true",
      "OCR_LLM_URL": "[https://api.anthropic.com/v1/messages](https://api.anthropic.com/v1/messages)",
      "OCR_LLM_TOKEN_KEY": "WORK_API_KEY",
      "OCR_LLM_MODEL": "claude-3-5-sonnet-20241022"
    }
  },
  "projects": {
    "ai-env": { "session": "019ec00f...", "provider": "opencode-go", "model": "minimax-m3" },
    "peer-1on1": "..."
  }
}

```

* `pi-projects.ts` でこの構造を読み込めるように型定義（`AiEnvConfig`, `ProfileConfig`, `ProjectConfig`）を追加し、パースとバリデーションを修正してください。
* （※ 古いフラットな形式の JSON が読み込まれた場合は、新しい構造に書き換えるようエラーメッセージで親切に案内し、パースを中断してOKです。）

### 2. プロファイルの自動判別 (`index.ts`)

ホスト側で実行された際のカレントディレクトリ（`process.cwd()`）のパス文字列を解析し、使用するプロファイルを自動決定してください。

* パスに `pi-private` が含まれていれば `"pi-private"` プロファイルを使用。
* パスに `pi-work` が含まれていれば `"pi-work"` プロファイルを使用。
* どちらも含まれない場合のデフォルトプロファイルも適切に設定するか、エラーにしてください。

### 3. Docker環境変数への動的マッピング (`index.ts`)

現在 `index.ts` の上部に定数としてハードコードされている `OCR_LLM_URL` や `OCR_LLM_MODEL` を削除してください。
`buildEnvArgs` 関数を修正し、自動判別されたプロファイル設定（`ProfileConfig`）を受け取り、以下の環境変数を Docker に注入してください。

* `--env=OCR_USE_ANTHROPIC=${profile.OCR_USE_ANTHROPIC}`
* `--env=OCR_LLM_URL=${profile.OCR_LLM_URL}`
* `--env=OCR_LLM_MODEL=${profile.OCR_LLM_MODEL}`
* `--env=OCR_LLM_TOKEN=${credentials[profile.OCR_LLM_TOKEN_KEY]}` (※ `CREDENTIAL_SOURCES` から取得した任意の値を選択して注入)

上記を満たすように、`RunContext` の型と `prepareEnvironment` などのデータの流れを整理し、必要な修正を適用した完全な `index.ts` と `pi-projects.ts` を出力してください。
