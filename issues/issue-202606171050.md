現在開発中の CLI ツール `ai-env` の `index.ts` において、環境変数の渡し忘れのバグがあるため修正をお願いします。

### 問題の背景
元のシェルスクリプトでは、コンテナ起動時に以下のように複数の環境変数を渡していました。
`-e XIAOMI_TOKEN_PLAN_SGP_API_KEY="..." -e OPENCODE_API_KEY="..." -e OPENROUTER_API_KEY="..." -e GH_TOKEN="..." -e OCR_LLM_TOKEN="..."`

現在の `index.ts` の `buildEnvArgs` 関数を確認すると、`OCR_LLM_TOKEN` にはプロファイル設定経由で正しく値が注入されていますが、**`OPENCODE_API_KEY` そのものを環境変数としてコンテナに渡す処理が抜け落ちています。**
（`CREDENTIAL_SOURCES` には定義されており、値自体は取得できています）

### 修正要件
`index.ts` の `buildEnvArgs` 関数を修正し、戻り値の配列に `OPENCODE_API_KEY` を追加してください。

追加する文字列のイメージ:
`--env=OPENCODE_API_KEY=${params.credentials.OPENCODE_API_KEY}`

その他の厳格なエラーハンドリング（`requireEnv` によるチェックや、クレデンシャル取得失敗時のエラー終了など）は現在の仕様のままで問題ありません。

対象ファイル: `index.ts`
