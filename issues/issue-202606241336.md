# 課題解決プロンプト: Apple Container 移行に伴う host.docker.internal の代替手段を実装する

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: コンテナを Docker から Apple Container に移行したところ、`host.docker.internal` というホスト名が使えなくなった。Docker では自動的にホストマシンの IP アドレスに解決される特殊なホスト名だったが、Apple Container にはこの機能がない。
- **期待される挙動**: Apple Container 環境でもホストマシンの IP アドレスを取得し、コンテナ内からホストのサービス (herdr.sock への socat 接続など) にアクセスできるようにする。
- **関連ファイル/コード**:
  - `index.ts`: `host.docker.internal` を使用している箇所
  - `pi-projects.ts`: `buildInitScript` 関数内の socat コマンドで `host.docker.internal` を使用
  - `Dockerfile`: `host.docker.internal` を使用している箇所があるか確認

## 3. 解決すべきゴール (Goal & Objective)
- [ ] ホストマシンの IP アドレスを動的に取得する方法を実装する
- [ ] `host.docker.internal` の代わりにホスト IP を使用するように修正する
- [ ] Docker 環境と Apple Container 環境の両方で動作する互換性を維持する (可能な場合)
- [ ] `npx tsc` / `npx oxlint` / `npx tsx --check` がすべてパスすることを確認する

### 制約事項
- Apple Container では `host.docker.internal` が使えないため、別の方法でホスト IP を取得する必要がある
- ホスト IP の取得方法は以下のいずれかを検討:
  - **方法 A**: macOS の `networksetup` コマンドや `ifconfig` でホスト IP を取得し、コンテナに環境変数として渡す
  - **方法 B**: `docker run` 時に `--add-host=host.docker.internal:host-gateway` 相当のオプションを使う (Apple Container で対応しているか確認)
  - **方法 C**: ホスト側でループバックアドレス `127.0.0.1` を使う (コンテナからホストへの接続が可能な場合)
  - **方法 D**: `host.docker.internal` を環境変数で上書き可能にする (ユーザが任意の IP を指定できるようにする)
- socat コマンドの接続先 `host.docker.internal:9123` を動的に変更できるようにする

### 実装のヒント
- `index.ts` でホスト IP を取得し、コンテナに `--env=HOST_IP=<ip>` で渡す
- `pi-projects.ts` の `buildInitScript` で socat の接続先を `$HOST_IP` に変更:
  ```bash
  # 変更前:
  pm2 start socat --name "herdr-socat" -- UNIX-LISTEN:/home/pi/.config/herdr/herdr.sock,fork,reuseaddr TCP:host.docker.internal:9123
  
  # 変更後:
  pm2 start socat --name "herdr-socat" -- UNIX-LISTEN:/home/pi/.config/herdr/herdr.sock,fork,reuseaddr TCP:${HOST_IP}:9123
  ```
- ホスト IP の取得方法 (macOS):
  ```typescript
  import { execFileSync } from "node:child_process";
  
  const getHostIp = (): string => {
    // macOS の en0 インターフェースの IP を取得
    const result = execFileSync("ipconfig", ["getifaddr", "en0"], { encoding: "utf8" });
    return result.trim();
  };
  ```
- または `scutil --dns` や `networksetup -getinfo "Wi-Fi"` でも取得可能
