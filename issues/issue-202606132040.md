`pi-projects.ts` の `buildInitScript` 関数が生成する初期化スクリプトにバグがあり、コンテナ起動後に `pi-resume` コマンドを実行すると `command not found` になってしまいます。

原因は、スクリプト内で関数を直接定義した直後に `exec /bin/bash` を実行しているため、新しいインタラクティブシェルに関数の定義が引き継がれないためです。

`buildInitScript` 関数を修正し、`pi-resume` 関数の定義をコンテナ内の `~/.bashrc` に追記するように変更してください。

具体的には、生成するスクリプトの最後を以下のようにヒアドキュメントを使って `.bashrc` に書き込む形にしてください。

```bash
# pi セッション再開用コマンドを .bashrc に追記
cat << 'EOF' >> ~/.bashrc
${piResumeFunc}
EOF

exec /bin/bash
```
この修正を適用した pi-projects.ts のコードを出力してください。
