# 課題解決プロンプト: Dockerfile のベストプラクティス対応（Node LTS版使用・バージョン固定・ARGキャッシュバスティング）

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `Dockerfile` に以下の1つのベストプラクティス違反がある：
  1. **ENV によるキャッシュバスティング**: `ENV CACHE="20260623"` — Docker の標準的なキャッシュバスティング手法は `ARG` + `--build-arg` であり、`ENV` を使うのはアンチパターン（環境変数としてコンテナ内に残り続ける）。
- **2026-06 時点で Node.js 24 は Active LTS** であり、Node 22 へのダウングレードは誤り。`FROM node:24-trixie-slim` を維持する。
- **@latest のバージョン固定はスキップ**: ユーザー判断により `playwright@latest` / `pm2@latest` を維持。playwright はバージョン固定すると依存解決の兼ね合いでビルドが失敗する場合があるため。
- **再現手順**: `Dockerfile` の各該当行を確認する。
- **関連ファイル/コード**:
  - `Dockerfile` 全体
  - `package.json`（engines フィールドの Node バージョン指定との整合性）

## 3. 解決すべきゴール (Goal & Objective)
- [x] `FROM node:24-trixie-slim` (Node 24 Active LTS) を維持
- [ ] npm グローバルインストール: ユーザー判断により `playwright@latest` / `pm2@latest` を維持（バージョン固定しない）
- [x] `ENV CACHE="..."` → `ARG CACHEBUST` に変更し、`docker build --build-arg CACHEBUST=$(date +%s)` で制御できるようにする
- [x] 既存のコンテナ起動・動作に影響がないことを確認する

## 4. 期待する成果物 (Output Format)
- 修正された `Dockerfile`
- ビルド確認結果（`container build -t pi-sandbox .` が成功すること）
- 動作確認結果（`ai-env` でコンテナが正常起動すること）
