私専用のAI開発用Dockerサンドボックス環境を簡単に起動するためのCLIツール「ai-env」を開発してください。

以下の要件を満たす `package.json` と `index.ts`（メインスクリプト）を作成してください。

# 技術スタック
- 言語: TypeScript
- パッケージ管理: npm
- 実行: tsx (ビルド不要で直接実行させる)
- CLIフレームワーク: commander.js

# コア機能と要件
macOSホスト上で実行され、動的にクレデンシャルを取得して以下の複雑な `docker run` コマンドを組み立て、子プロセスとして実行するCLIです。

1. **クレデンシャルの動的取得**:
   Node.jsの `child_process.execSync` を使い、以下のコマンドの標準出力を変数に格納してください（エラー時は空文字にするなど適切にハンドリング）。
   - XIAOMI_TOKEN_PLAN_SGP_API_KEY: `security find-generic-password -s "XIAOMI_TOKEN_PLAN_SGP_API_KEY" -w`
   - OPENCODE_API_KEY: `security find-generic-password -s "OPENCODE_API_KEY" -w`
   - OPENROUTER_API_KEY: `security find-generic-password -s "OPENROUTER_API_KEY" -w`
   - GH_TOKEN: `gh auth token`

2. **環境変数のマッピング**:
   取得したクレデンシャルと現在のプロセス環境変数を、Docker起動時の `-e` オプションにマッピングします。
   - `HERDR_PANE_ID` は現在の環境変数から引き継ぎ。
   - `OCR_LLM_URL` は "https://opencode.ai/zen/go/v1" に固定。
   - `OCR_LLM_TOKEN` は取得した `OPENCODE_API_KEY` を使用。
   - `OCR_LLM_MODEL` は "mimo-v2.5-pro" に固定。

3. **ボリュームマウントの設定**:
   - `$(pwd):/workspace` (現在のディレクトリ)
   - `~/.ssh:/tmp/.ssh:ro` (ホストのSSHディレクトリをRead-Onlyでマウント)
   - `~/.pi:/home/pi/.pi` (piの履歴永続化用)

4. **コンテナ起動と初期化スクリプト**:
   - ベースイメージ: `pi-private-sandbox`
   - エントリーポイント: `--entrypoint /bin/bash`
   - コンテナ起動時に `-c` で以下のスクリプトをワンライナーとして渡すこと:
     `cp -r /tmp/.ssh ~/.ssh && chown -R $(id -u):$(id -g) ~/.ssh && chmod 700 ~/.ssh && find ~/.ssh -type f -exec chmod 600 {} \; && mkdir -p ~/.config/herdr && socat UNIX-LISTEN:/home/pi/.config/herdr/herdr.sock,fork,reuseaddr TCP:host.docker.internal:9123 & exec /bin/bash`

5. **プロセスの実行**:
   組み立てたDockerコマンドを `child_process.spawnSync` で実行し、`stdio: 'inherit'` を指定してコンテナの標準入出力を現在のターミナルに直結させてください。
