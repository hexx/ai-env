# issue: pi-projects.json でコメントを使えるようにする

## 概要

`~/.config/ai-env/pi-projects.json` 設定ファイルで JSON コメント(`//` 行コメント、`/* */` ブロックコメント)を使えるようにする。現在は標準 `JSON.parse` のみでコメントを含むとパースエラーになる。

## 背景

設定ファイルが大きくなると、各プロファイル・プロジェクトの用途や、セッション ID の由来などをコメントで残したくなる。しかし標準 JSON はコメントをサポートしていないため、現在はコメントを書けない。

JSONC (JSON with Comments) は VS Code の `settings.json`、TypeScript の `tsconfig.json` などで広く使われており、ユーザの利便性を大きく向上する。

## 修正要件

### 1. コメント除去処理の追加 (`pi-projects.ts`)

`readFileSync` で読み込んだ文字列を `JSON.parse` に渡す前に、コメントを除去するステップを追加する。

対応すべきコメント形式:
- **行コメント**: `// ...` (行末まで)
- **ブロックコメント**: `/* ... */` (複数行対応)

**実装方針** (以下のいずれか):

#### 方針 A: `strip-json-comments` パッケージを使う (推奨)
- npm で `strip-json-comments` を dependencies に追加
- 厳密なパース(文字列内の `//` を誤って除去しない等のエッジケースに対応)
- メンテナンスされている安定パッケージ

```typescript
import stripJsonComments from "strip-json-comments";
// ...
const stripped = stripJsonComments(content);
return JSON.parse(stripped);
```

#### 方針 B: 独自の正規表現で除去する (依存なし)
- 新規依存を追加したくない場合の選択肢
- 正規表現で `//...` と `/*...*/` を除去
- 注意: 文字列内の `//` や `/*` を誤って除去しないよう、文字列リテラルを先に保護する必要がある

```typescript
const stripComments = (text: string): string =>
  text.replace(/\/\/.*$/gm, "").replace(/\/\*[\s\S]*?\*\//g, "");
```

### 2. テストの追加

以下のケースを検証するテストを作成する:
- 行コメントのみ含む設定ファイル → 正常にパースできる
- ブロックコメントのみ含む設定ファイル → 正常にパースできる
- 行コメントとブロックコメントの混在 → 正常にパースできる
- コメントが含まれない従来形式 → 従来通り動作する (後方互換)
- 文字列値の中に `//` を含む場合 → 文字列内の `//` は除去されない

### 3. ドキュメントの更新

`README.md` と `pi-projects.example.json` にコメント使用例を追加する:

