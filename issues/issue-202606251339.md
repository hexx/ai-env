# 課題解決プロンプト: pi-projects.json が未設定でも profiles のデフォルトで起動するモードを実装する

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: 新規プロジェクトや、`~/.config/ai-env/pi-projects.json` に記述がない場合、`ai-env` はエラーで終了する。初回利用時やテスト目的で素早くコンテナを起動したい場合に不便。
- **期待される挙動**: 設定ファイルが存在しない、または projects が空の場合でも、profiles のデフォルト設定でコンテナを起動できるモードを提供する。
- **関連ファイル/コード**:
  - `pi-projects.ts`: `loadAiEnvConfig` 関数（設定読み込み）、`AiEnvConfig` 型
  - `index.ts`: `prepareEnvironment` 関数（設定適用）、CLI オプション定義

## 3. 解決すべきゴール (Goal & Objective)
- [ ] 設定ファイルが存在しない場合のデフォルト動作を実装する
- [ ] profiles のデフォルト値（OCR 設定等）でコンテナを起動できる
- [ ] projects が空の場合はデフォルトプロジェクト（session: ランダム生成、provider/model: 任意）で起動する
- [ ] 既存の設定がある場合は従来通り動作する（後方互換）
- [ ] `npx tsc` / `npx oxlint` / `npx tsx --check` がすべてパスする

### 制約事項
- 既存の設定ファイルがある場合の動作を変更しない（後方互換）
- デフォルトプロファイルは `pi-private` を使用する（存在しない場合は最初のプロファイル）
- デフォルトプロジェクトは `default` という名前で作成し、session ID は自動生成する
- ユーザに明示的なフィードバックを与える（「デフォルト設定で起動します」等のメッセージ）

### 実装のヒント
- `loadAiEnvConfig` 関数を修正し、ファイル不存在時のエラーをデフォルト値生成に変更:
  ```typescript
  try {
    const content = readFileSync(configPath, "utf8");
    // 既存のパース処理
  } catch (error) {
    if (error.code === "ENOENT") {
      // デフォルト AiEnvConfig を返す
      return getDefaultConfig();
    }
    throw error;
  }
  ```
- デフォルト設定の生成:
  ```typescript
  const getDefaultConfig = (): AiEnvConfig => ({
    profiles: {
      "default": {
        OCR_USE_ANTHROPIC: "false",
        OCR_LLM_URL: "",
        OCR_LLM_TOKEN_KEY: "",
        OCR_LLM_MODEL: "",
      }
    },
    projects: {
      "default": {
        session: crypto.randomUUID(),
      }
    }
  });
  ```
- `prepareEnvironment` でデフォルト使用時のメッセージを表示:
  ```typescript
  if (isDefaultConfig) {
    console.error("設定ファイルが見つからないため、デフォルト設定で起動します。");
  }
  ```
