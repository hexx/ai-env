`pi-resume` 実行時のコマンドにおいて、設定ファイル (`pi-projects.json`) から `provider` と `model` を読み込んで適用し、さらに常に `--thinking high` オプションを付与するように修正してください。

以下の要件に従って、`pi-projects.ts` と `index.ts` を修正してください。

### 1. 型定義の追加と変更 (`pi-projects.ts`)
設定ファイルの値として、セッションIDの文字列だけでなく、オブジェクトも受け付けるように変更します。
`pi-projects.ts` に以下のインターフェースを追加し、`loadPiProjects` などの関数の戻り値や引数の型を `Record<string, string>` から `Record<string, ProjectConfig>` に変更してください。

```typescript
export interface ProjectConfig {
  session: string;
  provider?: string;
  model?: string;
}
```

### 2. バリデーションとパース処理の修正 (`pi-projects.ts`)

`validateAndCollect` 等の処理を修正し、以下の仕様を満たすようにしてください。

* `value` が従来通り「文字列」だった場合は、`{ session: value }` というオブジェクトに変換して扱う（後方互換性の維持）。
* `value` が「オブジェクト」だった場合は、`session` プロパティ（必須の非空文字列）を検証し、`provider` と `model`（いずれも任意、設定されていれば文字列）を受け入れる。

### 3. コマンド組み立ての修正 (`pi-projects.ts` の `generatePiResumeFunc`)

`generatePiResumeFunc` 関数を修正し、プロジェクトごとの case 文で生成するコマンドを以下のように組み立ててください。

* 常に `--thinking high` を付与する。
* 設定に `provider` があれば `--provider <provider>` を追加する。
* 設定に `model` があれば `--model <model>` を追加する。
* 最後に `--session <session>` を付与する。

生成されるコマンドのイメージ:
`pi --provider opencode-go --model minimax-m3 --thinking high --session 019ec00f...`

### 4. `index.ts` への変更の反映

`pi-projects.ts` の型の変更に伴い、`index.ts` 側でも `RunContext` インターフェース内の `piProjects` の型を `Record<string, ProjectConfig>` に修正し、必要な import を追加してください。
