# 課題解決プロンプト: profiles に apiKeyEnv 共通設定を追加

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `pi-projects.json` の `profiles` セクションには OCR 全体設定 (`OCR_*`) と `provider` / `model` のフォールバック値のみがあり、`apiKeyEnv` を共通設定として定義できない。同じ `apiKeyEnv` を複数の `projects` で使い回したい場合、各プロジェクトに同じ値を書かなければならず冗長。
- **既存のパターン**: `provider` / `model` はすでに `ProfileConfig` 側の値が `ProjectConfig` 側の値が無いときのフォールバックとして機能している。`pi-projects.ts` の `generatePiResumeFunc` 内で `config.provider ?? defaultProvider` / `config.model ?? defaultModel` という形で実装済み。
- **関連ファイル/コード**:
  - `/workspace/pi-projects.ts`
    - `interface ProfileConfig` (現状: `OCR_USE_ANTHROPIC`, `OCR_LLM_URL`, `OCR_LLM_TOKEN_KEY`, `OCR_LLM_MODEL`, `provider?`, `model?`)
    - `parseProfileOptionalFields` (現状: `provider` / `model` のみ SAFE_SHELL_PATTERN で検証)
    - `generatePiResumeFunc` のシグネチャと `apiKeyFlag` 組み立て処理
    - `buildInitScript` の引数 (現状: `defaultProvider` / `defaultModel` のみ受け取る)
    - `loadAiEnvConfig` (呼び出し側でデフォルト値を渡している箇所)
  - `/workspace/pi-projects.example.json` (ドキュメント兼サンプル)

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `ProfileConfig` インターフェースに `apiKeyEnv?: string` を追加する
- [ ] `parseProfileOptionalFields` で `apiKeyEnv` を `SAFE_ENV_NAME_PATTERN` (POSIX 環境変数名パターン) で検証する
- [ ] `generatePiResumeFunc` のシグネチャに `defaultApiKeyEnv: string | undefined` を追加し、`apiKeyFlag = config.apiKeyEnv ? \`--api-key "$${config.apiKeyEnv}"\` : defaultApiKeyEnv ? \`--api-key "$${defaultApiKeyEnv}"\` : ""` のフォールバックを実現する
- [ ] `buildInitScript` の引数に `defaultApiKeyEnv` を追加し、`generatePiResumeFunc` へ渡す
- [ ] `loadAiEnvConfig` から `buildInitScript` を呼ぶ箇所 (main 含む) で `defaultApiKeyEnv` を導出して渡す。導出方法: 設定ファイル先頭の profile の `apiKeyEnv` をデフォルトとする (もしくは `loadAiEnvConfig` の戻り値構造にデフォルト profile 名を含め、呼び出し側で参照する) いずれの設計が妥当か判断し実装する
- [ ] `pi-projects.example.json` の `pi-work` プロファイルに `apiKeyEnv` のサンプルを追加し、コメントで「プロジェクト側未指定時のフォールバック」であることを明記する
- [ ] 既存テスト (`*.test.ts`) すべて pass を維持する。新規追加された挙動に対するテストも追加する (例: プロジェクト側で `apiKeyEnv` 未指定 → profile 側の値が `--api-key` フラグに反映されるケース)
- [ ] `oxlint` を実行し、警告ゼロを確認する

## 4. 期待する成果物 (Output Format)
