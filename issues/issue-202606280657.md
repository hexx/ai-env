# 課題解決プロンプト: getCredential と getHostIp の encoding 指定不一致を解消する

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `index.ts` 内の2つの関数で `execFileSync` の使い方に一貫性がない：
  - `getHostIp`: `encoding: "utf8"` を指定 → 戻り値は `string`、`.trim()` のみ
  - `getCredential`: `encoding` 未指定 → 戻り値は `Buffer`、`.toString().trim()` が必要
  - この不一致により、コードを読んだ開発者が「なぜ片方だけ encoding 指定があるのか」と混乱する。意図的なのか単なる見落としなのか判別できない。
- **再現手順**: `index.ts` の `getHostIp` 関数と `getCredential` 関数の実装を比較すると確認できる。
- **関連ファイル/コード**:
  - `index.ts`: `getHostIp` 関数（L70-86付近）、`getCredential` 関数（L92-101付近）

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `getCredential` にも `encoding: "utf8"` を指定し、`.toString()` の冗長呼び出しを削除する
- [ ] または両関数で共通のラッパー関数（`execFileSync` を `encoding: "utf8"` 付きで呼ぶヘルパー）を導入する
- [ ] 既存の動作が変わらないことを確認する（クレデンシャル取得の挙動が同一であること）

## 4. 期待する成果物 (Output Format)
- 修正された `index.ts` の差分
- 動作確認結果（macOS 環境で `getCredential` が正しく動作すること）
