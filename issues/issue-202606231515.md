# 課題解決プロンプト: Dockerfile で RUN pi update を USER pi の前に移動し、キャッシュバスティング用 ENV を追加する

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: 現在、`Dockerfile` で `RUN pi update` が `USER pi` の後に実行されているが、`pi update` が動作しない。`pi update` は root 権限で実行する必要があるため、`USER pi` の前に移動する必要がある。
- **追加要件**: Docker のレイヤーキャッシュにより、`RUN pi update` が再実行されない問題がある。`ENV CACHE="20260623"` のような無意味な文字列を配置し、好きなタイミングで値を変更することで `docker build` 時に `pi update` が再実行されるようにする。
- **関連ファイル/コード**:
  - `Dockerfile`: `RUN pi update` と `USER pi` の順序、`ENV` の追加

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `RUN pi update` を `USER pi` の前に移動する
- [ ] `RUN pi update` の前に `ENV CACHE="20260623"` を追加する (キャッシュバスティング用)
- [ ] `pi update` が root 権限で実行されることを確認する
- [ ] `docker build` が正常に完了することを確認する

### 制約事項
- `RUN pi update` は root 権限で実行する必要がある
- `ENV CACHE` の値は任意の文字列でよい (日付やランダムな文字列など)
- `ENV CACHE` の値を変更すると、その行以降の Docker レイヤーがキャッシュバスティングされ、`RUN pi update` が再実行される

### 実装のヒント
- 現在の Dockerfile 構造:
  ```dockerfile
  USER pi
  RUN pi update
  ```
- 変更後の Dockerfile 構造:
  ```dockerfile
  ENV CACHE="20260623"
  RUN pi update
  USER pi
  ```
- `ENV CACHE` の値を変更すると、Docker はその行以降のレイヤーを再構築するため、`RUN pi update` が再実行される

- 修正が完了した `Dockerfile`
- 修正理由とアプローチの簡潔な解説
- `docker build` の動作確認結果
