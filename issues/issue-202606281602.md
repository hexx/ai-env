# 課題解決プロンプト: テストファイルをモジュール単位に分割（pi-validation / pi-config / pi-script / index）

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: ソースコードを `pi-types.ts` / `pi-validation.ts` / `pi-config.ts` / `pi-script.ts` の4モジュールに分割したリファクタリングが完了しているが（PR #52）、**テストファイルは依然として `pi-projects.test.ts` 一本に631行が集約されたまま**。これにより以下の問題が生じている：
  1. 1つのテストファイルに複数のモジュールのテストが混在し、目的のテストを探すのが困難
  2. ソース分割と整合せず、リファクタリングの意図がテストレイヤで反映されていない
  3. 将来的に `index.test.ts` を追加するとさらに肥大化する
- **再現手順**: `ls *.test.ts` を実行すると `pi-projects.test.ts` 1ファイルのみが表示される。
- **関連ファイル/コード**:
  - `pi-projects.test.ts`（631行、6 describe ブロック、32 it ブロック）
  - `package.json` の `"test"` スクリプト: `node --import tsx --test pi-projects.test.ts`

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `pi-projects.test.ts` の describe ブロックを、対応するソースモジュール名のテストファイルに分割する：
  - `pi-validation.test.ts`: `validateCliOverrides` 関連
  - `pi-config.test.ts`: `ProfileConfig.apiKeyEnv` 関連（loadAiEnvConfig 経由）
  - `pi-script.test.ts`: `buildInitScript` 関連（3 describe ブロック：デフォルト起動 / CLI オーバーライド / --bash モード / apiKeyEnv フォールバック）
- [ ] `package.json` の `"test"` スクリプトを `node --import tsx --test *.test.ts` に変更し、全テストファイルが自動実行されるようにする
- [ ] 既存の32テストケースが全て新ファイルに移植され、引き続き通過すること
- [ ] 旧 `pi-projects.test.ts` は削除する

## 4. 期待する成果物 (Output Format)
- 新規作成された4つのテストファイル
- 修正された `package.json` の diff
- 削除された `pi-projects.test.ts`
- テスト実行結果（合計32件以上、全件通過）

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、直ちにステップ2の「関連ファイル」の解析から始めてください。
