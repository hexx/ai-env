# 課題解決プロンプト: generateCaseBody の unknownStrategy 文字列ユニオンを改善する

あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `pi-script.ts` の `generateCaseBody` 関数が `unknownStrategy: "warn" | "defaults"` という文字列リテラルユニオンを引数に取り、`if (unknownStrategy === "warn")` のような文字列比較で分岐している。これは実質的に2択の boolean フラグであり、以下の問題がある：
  1. 新しい戦略が増える見込みがない中で文字列ユニオンを使うのは過剰設計
  2. 文字列比較による制御フローはタイポ耐性が低く、IDE のリファクタリング支援も受けにくい
  3. 関数の関心が2つの異なる振る舞い（警告 vs. デフォルト起動）に分散している
- **再現手順**: `pi-script.ts` の `generateCaseBody` 関数のシグネチャと実装を確認する。
- **関連ファイル/コード**:
  - `pi-script.ts`: `generateCaseBody` 関数
  - `pi-script.ts`: `generatePiResumeFunc`（`unknownStrategy: "warn"` で呼び出し）
  - `pi-script.ts`: `buildInitScript`（`unknownStrategy: "defaults"` で呼び出し）

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `unknownStrategy` を削除し、より意図が明確な形にリファクタリングする。選択肢：
  - 2つの別関数に分割する（`generateCaseBodyForResume` / `generateCaseBodyForDefaults`）
  - boolean フラグ（`warnOnUnknown: boolean`）に置き換える
- [ ] 呼び出し側（`generatePiResumeFunc`, `buildInitScript`）も合わせて修正する
- [ ] 既存の全テストが通過すること
