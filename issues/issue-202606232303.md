# 課題解決プロンプト: ai-env コマンドに bash を起動するオプションを追加する

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: 現在、`ai-env` コマンドはコンテナ起動時に自動的に `pi` (pi-coding-agent) を起動する。しかし、デバッグやトラブルシューティング時には `pi` を起動せずに bash シェルのみで操作したい場合がある。
- **期待される挙動**: `ai-env --bash` オプションを指定すると、コンテナ内で `pi` を起動せずに bash シェルのみを起動する。
- **関連ファイル/コード**:
  - `index.ts`: CLI オプション定義、`buildDockerArgs` 関数
  - `pi-projects.ts`: `buildInitScript` 関数 (初期化スクリプト生成)

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `ai-env` コマンドに `--bash` オプションを追加する
- [ ] `--bash` オプション指定時、コンテナ内で `pi` を起動せずに bash シェルのみを起動する
- [ ] `--bash` オプション未指定時は従来通り `pi` を起動する
- [ ] `npx tsc` / `npx oxlint` / `npx tsx --check` がすべてパスすることを確認する

### 制約事項
- `--bash` オプションは `commander` のオプションとして追加する
- `--bash` オプション指定時、`buildInitScript` で生成する初期化スクリプトの末尾を `exec /bin/bash` に変更する
- `--bash` オプション未指定時は従来通り `pi-resume` + `pm2 kill` + `exit` のフローを維持する

### 実装のヒント
- `index.ts` に `--bash` オプションを追加:
  ```typescript
  program
    .option("--bash", "pi を起動せずに bash シェルのみを起動する")
  ```
- `RunContext` に `bashMode: boolean` を追加
- `buildInitScript` に `bashMode` パラメータを追加し、条件分岐で末尾を変更:
  ```typescript
  if (bashMode) {
    return script + "\nexec /bin/bash";
  } else {
    return script + "\n${piResumeFunc}\npi-resume\nrc=$?\npm2 kill\nexit $rc";
  }
  ```
- `buildDockerArgs` に `bashMode` を渡す
